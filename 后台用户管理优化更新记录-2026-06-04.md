# 后台用户管理优化更新记录 - 2026-06-04

## 本次更新

- 修复后台账号首次注册后仍显示未激活的问题：
  - 注册激活时显式更新 `password`、`status=active`、`nickname`。
  - 更新后重新查询账号再返回，避免前端拿到旧状态。
  - 首个后台账号 bootstrap 仍允许注册为超级管理员。
- 后台账号权限模型简化：
  - 不再使用权限等级控制页面访问。
  - 新增 `permissions` 页面权限字段，直接存储允许访问的页面列表。
  - 非管理员账号即使传入 `users` 页面，也会在后端过滤。
- 后台身份调整：
  - 身份固定为运营 `operation`、客服 `customer_service`、管理员 `admin`。
  - 管理员判断只看 `role=admin`，不再看 `level`。
  - 仍防止移除或停用最后一个启用状态管理员。
- Web 账号管理页优化：
  - 新增/编辑账号时可直接勾选页面权限。
  - 新增账号默认待激活，避免创建无密码的启用账号。
  - 用户列表展示身份、状态和可访问页面。
