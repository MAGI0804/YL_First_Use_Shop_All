# Redis缓存与后台登录更新记录 - 2026-06-03

## 本次更新

- 后端活动图读取增加 Redis 缓存：
  - `GetActivityImageByID` 增加详情缓存。
  - `QueryActivityImages` 增加列表缓存，缓存键包含分页、状态、时间范围和活动详情筛选条件。
  - 活动图新增、上下线、下线、排序、详情/宣传图更新后统一清理 `cache:activity:*`。
  - Redis 不可用时自动走数据库，不影响业务请求。
- 后台 Web 登录流程改为真实账号体系：
  - 支持手机号 + 密码登录。
  - 支持首次登录时手机号 + 密码 + 验证码激活账号。
  - 验证码继续使用 `service/sms` 包发送，并复用 Redis 验证码存储。
  - 后台 token 使用 JWT，前端请求通过 `Authorization: Bearer <token>` 校验当前账号。
- 后台账号管理：
  - 管理员可新增待激活手机号和账户名。
  - 管理员可查看账号列表和用户状态。
  - 管理员可切换账号 `pending`、`active`、`disabled`。
  - 防止停用最后一个 active 管理员。
  - 当后台账号表为空时，允许一次性注册第一个超级管理员；之后必须由管理员预添加手机号。
- Web 端权限：
  - 登录后保存后台 token 和账号信息。
  - 路由进入后台页面前调用 `backend_me` 校验 token 和账号状态。
  - 左侧菜单按后端返回的 `permissions` 显示。
  - 管理员账号显示“账号管理”菜单。
