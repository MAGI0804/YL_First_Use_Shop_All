# 商品图片加载优化更新记录-2026-06-05

## 项目检查

- 当前项目包含 `Member_Shop` Go 后端、`Management_web_shop` Vue 管理端、`wx_ui_kids` 微信小程序端。
- 后端商品列表接口已按列表场景返回 `promo_image_url`，没有返回详情长图；上传工具当前为原图保存，未生成缩略图。
- 小程序商品详情页会把主图和展示长图全部放入页面图片节点，图片较大时容易造成内存占用和加载变慢。
- 小程序 `lazy-image` 组件原先使用 `wx.getImageInfo` 预取图片，再赋值给 `<image>`，会造成额外下载和解码。

## 本次更新

1. 更新 `wx_ui_kids/components/lazy-image`：
   - 移除 `wx.getImageInfo` 预取逻辑。
   - 图片进入视口后直接切换真实地址，避免重复下载和重复解码。
   - 首次触发后断开观察器，减少滚动过程中的观察器开销。

2. 更新 `wx_ui_kids/pages/commodity/goods`：
   - 详情轮播新增 `goodsImageItems`，只让当前图和相邻图使用真实地址。
   - 未进入当前/相邻范围的轮播图先显示占位图，减少进入详情页时同时解码的大图数量。
   - 商品展示长图、规格弹层图补充小程序原生 `lazy-load`。

3. 更新 `Management_web_shop` 商品图片展示：
   - 商品列表和商品详情图片补充 `loading="lazy"`、`decoding="async"`。
   - 降低管理端列表/详情页同时解码多张商品图的压力。

- 已执行：`node --check wx_ui_kids/pages/commodity/goods/index.js`，通过。
