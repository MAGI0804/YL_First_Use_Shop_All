# 启动和部署说明

## 1. 后端 `Member_Shop`

1. 复制环境变量模板：

```powershell
Copy-Item Member_Shop\.env.example Member_Shop\.env
```

2. 按部署环境填写 `.env`，生产环境必须设置：

- `DB_HOST`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`
- `REDIS_ADDR`、`REDIS_PASSWORD`
- `JWT_SECRET`
- `WECHAT_APP_ID`、`WECHAT_APP_SECRET`
- `ALIYUN_SMS_*`
- `JST_*`
- `CORS_ALLOW_ORIGINS`

3. 启动：

```powershell
cd Member_Shop
go test ./...
go run .
```

默认端口是 `3088`，可通过 `APP_PORT` 覆盖。日志目录默认 `./logs`，可通过 `LOG_DIR` 覆盖。

## 2. Web 管理端 `Management_web_shop`

1. 复制环境变量模板：

```powershell
Copy-Item Management_web_shop\.env.example Management_web_shop\.env
```

2. 设置后端 API 地址：

```text
VITE_API_BASE_URL=http://127.0.0.1:3088
```

3. 本地启动或构建：

```powershell
cd Management_web_shop
npm install
npm run build
npm run dev
```

构建产物生成在 `Management_web_shop/dist`。

## 3. 小程序 `wx_ui_kids`

请求层会按微信环境自动选择 API 地址：

| 微信环境 | API 地址 |
| --- | --- |
| `develop` | `http://127.0.0.1:3088` |
| `trial` | `https://shop-test.youlankids.com` |
| `release` | `https://snow-api.youlankids.com` |

本地联调可在调试器里设置：

```js
wx.setStorageSync('api_host_override', 'http://127.0.0.1:3088')
```

清理覆盖：

```js
wx.removeStorageSync('api_host_override')
```

## 4. 上线检查

- `Member_Shop/.env` 不提交真实密钥。
- `JWT_SECRET` 使用长随机字符串。
- `CORS_ALLOW_ORIGINS` 在生产环境只填写实际 Web 管理端域名。
- 确认 MySQL、Redis、日志目录具备读写权限。
- 上线前执行 `go test ./...` 和 `npm run build`。
