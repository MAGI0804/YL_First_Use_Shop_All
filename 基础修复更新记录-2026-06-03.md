## 更新内容

### 第 2 到 3 周：订单和统计口径

- 数据分析销售统计改为按已支付订单的 `final_pay_amount` 汇总实际销售额。
- 报表接口新增 `original_order_amount` 和 `discount_amount`，用于保留原订单金额与优惠金额对账口径。
- 日销售明细同步使用 `final_pay_amount` 计算 `sales_amount` 和 `paid_amount`。

### 第 4 到 5 周：库存基础闭环

- 库存日志类型按开发文档统一为：
  - `order_create_deduct`
  - `order_cancel_restore`
  - `return_completed_restore`
  - `manual_adjust`
  - `jushuitan_sync`
  - `stock_transfer`
  - `stock_check`
- 售后和取消订单库存回滚兼容历史 `order_deduct` 日志，避免旧数据无法回滚。
- 新增库存调拨接口：`POST /inventory/transfer`。
- 新增库存盘点接口：`POST /inventory/stock_check`。
- Web 管理端新增“库存管理”页面，包含库存查询、库存预警、库存日志、手动调整、库存调拨、库存盘点。
- Web 管理端新增库存接口类型和 API 调用封装。
