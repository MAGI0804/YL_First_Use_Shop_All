# 第六周基础修复更新记录（2026-06-03）

分支：`develop`

## 本次范围

- 第 6 周售后体系基础修复。

## 更新记录

### 第一批：后端售后统计与创建校验

- 新增售后统计接口：`POST /return_order/statistics`。
- 售后统计返回申请总数、待处理数、完成数、售后率、售后金额、售后原因排行。
- 售后金额按已完成售后单关联的子订单金额或订单最终实付金额汇总。
- 售后率按售后订单数 / 已完成订单数计算。
- 售后创建请求不再强制所有买家地址字段必填，避免“仅退款”被地址校验拦截。
- 新增售后统计时间解析与售后率单元测试。

### 第二批：Web 售后中心与小程序售后闭环

- Web 售后页面移除静态演示数据，接入 `POST /return_order/query` 查询真实售后单。
- Web 售后页面接入 `POST /return_order/approve`，支持运营通过或拒绝售后申请。
- Web 售后页面接入 `POST /return_order/receive`，支持运营确认收货并完成售后。
- Web 售后页面接入 `POST /return_order/statistics`，展示申请数、待审核数、完成数、售后率、售后金额和原因排行。
- 小程序售后申请改为调用 `POST /return_order/create`，提交退货、换货、仅退款和已选商品 ID。
- 小程序售后提交成功后跳转售后详情，展示售后审核进度。
- 小程序售后详情在审核通过后支持买家填写寄回物流，并调用 `POST /return_order/deliver`。
- 小程序寄回物流提交增加登录用户校验，避免缺失 `user_id` 时提交无效请求。
